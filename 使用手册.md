# 小敏电商主图设计 Skill 使用手册

## 1. 这个 Skill 是做什么的

`xiaomin-ecommerce-main-image-skill` 用于设计、规划、生成和复盘电商主图。它适合中国电商语境下的商品主图，包括天猫、淘宝、京东、抖音电商、拼多多、小红书、Amazon、Shopee、Lazada 等平台。

它的核心不是简单“做一张好看的图”，而是先确认商品和交易信息，再决定主图结构。因为主图是否展示价格、券、赠品、会员机制、活动时间、物流承诺，会直接影响画面布局和合规风险。

## 2. 适用场景

可以用于：

- 给产品图生成电商主图
- 把普通商品图改造成高点击主图
- 为活动主图整理价格、券、机制和利益点
- 分析主图为什么不好看、不转化
- 根据竞品/参考主图提炼设计原则
- 为 `gpt-image-2` 生成稳定的主图生图提示词
- 使用 Cowart 进行主图画布迭代和批注

不建议用于：

- 详情页长图设计
- 品牌官网首屏
- 海报、banner、KV，而不是商品主图
- 需要编造价格、功效、认证或活动机制的需求

## 3. 标准工作流

### 第一步：先整理确认信息

当你提出一个新主图需求时，skill 会先输出一张“信息确认卡”，通常包括：

```text
产品名称
品牌
类目
规格/SKU
目标平台/场景
主图类型判断
价格信息
活动/优惠机制
赠品/会员机制
活动时间
核心利益点
必须展示文案
不展示/禁用信息
素材质量判断
缺失信息
```

你确认后，skill 才继续做图或写提示词。

### 第二步：判断主图类型

常见类型包括：

- 白底搜索主图
- 场景点击主图
- 卖点主图
- 质地主图
- 对比主图
- 组合/SKU 主图
- 活动/价格券主图
- 赠品/会员机制主图
- 质感大特写 + 小包装主图

如果没有价格、优惠、赠品等信息，skill 不会主动做大价格栏或活动主图。

### 第三步：压缩主图文案

主图文案一般控制为：

- 1 个主标题
- 1 个副标题
- 1-3 个徽章或机制卡

例如：

```text
主标题：一口沉醉 乐享丝滑
服务 icon：冰袋冷运
```

或者：

```text
主标题：幼猫专研
副标题：亲肠胃 好消化
机制卡：入会建档享5元券
服务条：顺丰发货｜送货上门
```

### 第四步：生成或编辑主图

如果提供了产品图，优先使用产品图作为参考进行编辑/重构，尽量保持包装、Logo、颜色、SKU 和商品形态。

如果没有产品图，只能做方向稿，不能保证包装准确。

### 第五步：质量与合规检查

生成后会检查：

- 商品是否 1 秒内可识别
- 主标题是否可读
- 商品是否足够大
- 是否像正式电商主图，而不是海报或 PPT
- 是否乱加了价格、活动、赠品、认证、功效
- 是否存在 AI 乱码、Logo 跑偏、包装变形

## 4. 推荐使用方式

### 示例 1：普通卖点主图

```text
$xiaomin-ecommerce-main-image-skill
做一张德芙经典巧克力主图。
卖点：一口沉醉 乐享丝滑。
夏日专属 icon：冰袋冷运。
符合品牌调性。
```

skill 会先整理确认：

- 产品名称：德芙经典巧克力
- 价格：用户未提供，不展示价格
- 机制：冰袋冷运作为服务 icon
- 主图类型：品牌调性卖点主图 + 质地主图

确认后再做图。

### 示例 2：带会员机制主图

```text
$xiaomin-ecommerce-main-image-skill
做一张皇家幼猫粮主图。
卖点：幼猫专研，亲肠胃，好消化。
机制：入会建档享5元券。
服务：顺丰发货，送货上门。
```

推荐结构：

- 主标题：幼猫专研
- 副标题：亲肠胃 好消化
- 机制卡：入会建档享5元券
- 物流信任条：顺丰发货｜送货上门

### 示例 3：活动价格主图

```text
$xiaomin-ecommerce-main-image-skill
做一张饮料活动主图。
产品：满分 NFC 橙汁 280ml x 15 瓶。
价格：会员预估到手 83 元/件。
机制：跨店满 300 减 30，会员领券再减 5 元。
活动时间：4/24 20:00 - 4/28 23:59。
```

这类会进入活动利益型主图，允许使用：

- 顶部活动条
- 中部商品和卖点
- 底部价格/券/优惠栏

但所有价格和机制必须由用户提供。

## 5. 信息填写建议

为了让结果更稳，最好一次性提供：

```text
产品名称：
品牌：
产品图：
规格/SKU：
目标平台：
主推卖点：
价格：
优惠机制：
赠品/会员权益：
活动时间：
必须展示文案：
不能展示的信息：
风格参考：
```

如果某些信息没有，就明确写“无”或“不展示”。比如：

```text
价格：不展示
赠品：无
活动时间：不展示
```

## 6. 合规注意事项

不要让主图展示未确认的信息，包括：

- 未提供价格
- 未提供优惠券
- 未提供满减
- 未提供赠品
- 未提供会员权益
- 未提供活动时间
- 未提供平台官方标识
- 未提供认证/奖项
- 医疗或绝对化功效

对于食品、宠物、健康类商品尤其要谨慎：

- 不要写治疗、根治、改善疾病
- 不要写 100%、最强、第一
- 不要编造成分、认证、营养功效

## 7. 输出状态说明

生成结果通常分三类：

- `可作为方向稿`：结构和调性可用，但素材或包装准确度不足
- `接近正式稿`：可继续精修，商业完成度较高
- `不合格需重做`：出现严重乱码、乱加信息、商品变形或画面不成立

如果没有高清产品图，默认只能输出方向稿。

## 8. 安装方式

把 skill 文件夹复制到 skills 目录：

```bash
cp -R xiaomin-ecommerce-main-image-skill ~/.agents/skills/
```

然后使用：

```text
$xiaomin-ecommerce-main-image-skill 做一张产品主图
```

## 10. 最重要的一条

先确认信息，再做图。

尤其是价格、优惠机制、赠品、会员权益、活动时间和强利益点，必须确认后才能进入画面。这样主图才不会“看着像电商图，但实际不能用”。
